## 关于文档

### docsify使用说明
> 文档由docsify支撑

| 指令                    | 功能                                               |
| -------------          | --------------------------------------------------|
| npm i docsify-cli -g   | 安装依赖                                            |
| docsify init ./docs    | 初始化docs                                          |
| docsify serve docs     | 启动docs,实时预览                                    |
| docsify serve docs -p port  | 启动docs指定端口 `localhost:port `              |

更多文档说明:[docsify轻量级文档](https://docsify.js.org/#/?id=docsify)

## 目录解释

### cosmos框架

### 前端规范
> 统一的前端规范文档

### 项目文档

#### 文件目录
> 里面记录我们整个项目的目录树，层次比较分明，标注一下每个文件        的作用是什么。（有个mddir的工具可以帮助自动生成目录结构，避免手动去写，节省时间）
#### 依赖管理
> 记录我们前端项目的一些核心依赖项（React项目通用依赖可以不记录）以及引入他们的作用和原因及位置。

#### 路由配置
> 里面记录路由与文件、页面的映射关系。

#### 服务配置
> 服务端接口记录、axios配置等

#### 组件
> 核心组件、通用组件记录。这个在业务庞大或者逻辑复杂的项目中重要性更高一点。

### 开发注意事项
> 记录一些较重的业务逻辑等。

### 变更记录
> 里面记录此次迭代的新增功能、优化功能、新增接口、新增依赖、新增组件、开发者等信息。
> 或者底层框架升级的信息、前端做的设计评审内容。

### 其他事项
> 其他不属于以上部分的事情都可以记录在此。

# MOCK

> 李淳 @2020-07-15

前后端分离的开发模式，要求前端在开发时需自行mock数据，我们推荐使用代理的方式，实施调整，而非在代码中进行特殊处理。当前更多的采用的是**easyMock**服务[http://easy-mock.sftcwl.com](http://easy-mock.sftcwl.com)，后续会推进yapi的mock服务落地，进一步降低数据mock成本。

[EasyMock的使用文档](http://wiki.sftcwl.com/pages/viewpage.action?pageId=21827126)
